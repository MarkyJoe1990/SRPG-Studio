Selector Event Commands by MarkyJoe1990

How to use

1. Put the entire "Selector-Event-Commands" folder in your SRPG Studio plugin folder.

2. Click Database -> Config -> Variables -> Options and check the box that allows variables to be used in events

3. Click Tools -> Options -> Data and check the box that says "Display id next to data name"

4. In your events, create an "Execute Script" event command.

5. In the "Script" tab, click "Call Event Command" and put in the name of the event you want to call.

6. In the "Property" field, create open and closed curly braces "{}", and put whatever arguments specified by each event command in between the braces. Example:

{
	var1: "TargetItem",
	weaponList: [0,1,2,3],
	itemList: [0,1,2,3],
	selfSwitch: "A"
}

5. If you are using an event command that uses Original Data, go to that tab and set whatever fields that are being used.

Event Commands

Name Description Original Data Properties
Item List Select Select from a user-defined list of items. None var1, weaponList, itemList
Convoy Item Select Select an item from the convoy. None var1
Unit Item Select Select an item from a unit you specify. Unit var1
Skill List Select Select from a user-defined list of skills. None var1, skillList
Unit Skill Select Select a skill from a unit you specify. Unit var1
Class List Select Select from a user-defined list of classes. None var1, classList
Unit List Select Select from a user-defined list of player units. None var1, unitList
State List Select Select from a user-defined list of states. None var1, stateList

Properties

var1 : "Variable Name"

String. When the player makes a selection, the selection's ID is stored in the variable whose name you specify here. This property is required for all selector events. Using an ID variable is recommended

var2 : "Variable Name"

String. When the player makes a selection, the selection's position on the list is stored in the variable whose name you specify here. This property is optional for all selector events and currently does not do anything useful.

selfSwitch: "Letter"

String or Number. If the player cancels out the list selection, the self switch you specify will be turned on. If you don't see this argument, then the player cannot cancel out of the window.

weaponList : [Weapon ID, Weapon ID, Weapon ID, ...]

Array. Defines a list of weapons that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each weapon.

itemList : [Item ID, Item ID, Item ID, ...]

Array. Defines a list of items that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each item.

unitList : [Unit ID, Unit ID, Unit ID, ...]

Array. Defines a list of units that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each unit.

classList : [Class ID, Class ID, Class ID, ...]

Array. Defines a list of classes that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each class.

skillList : [Skill ID, Skill ID, Skill ID, ...]

Array. Defines a list of skills that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each skill.

stateList : [State ID, State ID, State ID, ...]

Array. Defines a list of states that will be included in this list. Make sure you did step 3 of "How To Use" so you can know the IDs of each state.
